Automation schedule

FrequencyScriptWhat it does
Every 15 min activity-watcher.py Detects new activities; triggers Claude analysis; stubs session-log.json; sends Telegram coaching note with inline keyboard for RPE, nutrition, and pain capture
Continuous bot.py Two-way Telegram bot; routes messages to Claude or fast-path handlers; processes inline keyboard callbacks (RPE, nutrition, pain scores, FTP/CSS/LTHR updates)
Daily 06:20 morning-checkin.py Fetches HRV, sleep, and wellness from Intervals.icu; sends personalised daily readiness note and today's session preview
Daily 20:00 evening-checkin.py End-of-day check-in; prompts for post-session notes if any activities occurred; reminds of tomorrow's session
Sunday 20:00 weekly-summary.py Fetches week's training data directly from Intervals.icu; generates compliance card with TSS, CTL change, sleep, fuelling; evaluates 8 decision triggers (recovery, overreach, phase transitions, injury, nutrition, HRV); sends to Telegram
Hourly refresh-site-data.py Fetches live training metrics from Intervals.icu; writes training-data.json powering the athlete dashboard
Hourly refresh-public-data.py Extracts public-safe fields (CTL trend, race countdown) from training-data.json; writes site-data.json and pushes to GitHub Pages
On demand generate-blueprint.py Generates the season training blueprint — phase structure, TSS ceilings, test schedule — and pushes test events to Intervals.icu calendar
On demand generate-race-plan.py Generates race-day targets from current FTP, CSS, LTHR, and forecast CTL; supports all triathlon distances; triggered via Telegram or manually

Athlete data & privacy

All personal athlete data (session logs, injury notes, health metrics, training analysis) is stored exclusively on the VM. The ClaudeCoach/athletes/ directory is gitignored — nothing personal is ever pushed to the public GitHub repository.

The public site receives only a minimal, anonymised summary (race countdown, CTL fitness trend) via site-data.json, which contains no health data, weight, injury details, or identifying performance metrics.

Onboarding

New athletes are invited by the admin via /invite <chat_id>. The bot walks them through a multi-step Telegram questionnaire: name, target race, Intervals.icu credentials — then immediately fetches FTP, thresholds, and fitness history from ICU and asks only for the gaps. The admin approves with /approve <slug>, which scaffolds the athlete folder, generates a personalised system prompt, and activates coaching.
